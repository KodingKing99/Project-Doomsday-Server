## Architecture

.NET 9 hexagonal/clean architecture. Dependency flow (outside-in): `WebApi ‚Üí Infrastructure ‚Üí Application ‚Üí Domain`

- **Domain** ‚ÄĒ Pure models. `File` entity, config POCOs (`MongoDbConfig`, `S3Config`).
- **Application** ‚ÄĒ Business logic. `IFilesService`/`FilesService` orchestrates operations. Ports: `IFileStorage`, `IFileRepository`.
- **Infrastructure** ‚ÄĒ Adapters: `MongoDbFileRepository`, `S3FileStorage`, `LocalFileStorage`, `InMemoryFileRepository`. DI via `AddInfrastructureServices()`.
- **WebApi** ‚ÄĒ ASP.NET Core entry point. `Program.cs` is the DI composition root.

## Key Patterns

- **Storage key format**: `{UserId}/{FileId}` ‚ÄĒ generated after the file record is created.
- **Presigned URL workflow**: `POST /files` creates metadata and returns a presigned S3 PUT URL. Client uploads directly to S3.
- **Pagination**: `GET /files?skip=0&take=50`. Take clamped to max 200. Sorted by `UpdatedAtUtc` descending.
- **Authentication**: Optional Cognito JWT Bearer, toggled via `Authentication:Enabled` in config.
- **All I/O is async** with `CancellationToken` threaded throughout.
- **File.Id** is nullable on creation (auto-generated by MongoDB's `StringObjectIdGenerator`).
- **REST endpoints**: Separate `POST /files` (create) and `PUT /files/{id}` (update) ‚ÄĒ not upsert.

## Test Patterns

**ApiTests** (`src/ProjectDoomsdayServer.ApiTests/`) ‚ÄĒ Fast, no Docker.
- `IClassFixture<CustomWebApplicationFactory>` with NSubstitute mocks for `IFileStorage` and `IFileRepository`.
- Factory tracks state via `SavedFiles`, `DeletedFileIds`, `FileRecords`. Call `factory.Reset()` between tests.

**E2ETests** (`src/ProjectDoomsdayServer.E2ETests/`) ‚ÄĒ Full-stack, requires Docker.
- `ICollectionFixture<E2EInfrastructureFixture>` (shared containers) + per-class `E2EWebApplicationFactory`.
- Testcontainers spins up MongoDB (`mongo:7.0`) and MinIO. Test isolation via `UniqueUserId()` ‚ÄĒ no teardown needed.
- `_appClient` ‚Üí in-process server; `_rawHttpClient` ‚Üí MinIO for presigned PUT uploads.

Assertions use FluentAssertions (`response.StatusCode.Should().Be(HttpStatusCode.OK)`).

**Test commands ‚ÄĒ use surgical output by default:**
```bash
# Mocked tests only ‚ÄĒ start here (no Docker required)
dotnet test --filter "FullyQualifiedName~ApiTests" --verbosity minimal 2>&1 | head -n 150

# E2E tests (requires Docker)
dotnet test --filter "FullyQualifiedName~E2ETests" --verbosity minimal 2>&1 | head -n 150

# Specific class or method
dotnet test --filter "FullyQualifiedName~FileCrudIntegrationTests" --verbosity minimal 2>&1 | head -n 100

# More context around failures (stack traces)
dotnet test 2>&1 | grep -A 20 "Failed\|Error\|Exception" | head -n 200

# Full output (last resort)
dotnet test --verbosity normal 2>&1
```
