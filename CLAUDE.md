## Architecture

.NET 9 hexagonal/clean architecture. Dependency flow (outside-in): `WebApi â†’ Infrastructure â†’ Application â†’ Domain`

- **Domain** â€” Pure models. `File` entity, config POCOs (`MongoDbConfig`, `S3Config`).
- **Application** â€” Business logic. `IFilesService`/`FilesService` orchestrates operations. Ports: `IFileStorage`, `IFileRepository`.
- **Infrastructure** â€” Adapters: `MongoDbFileRepository`, `S3FileStorage`, `LocalFileStorage`, `InMemoryFileRepository`. DI via `AddInfrastructureServices()`.
- **WebApi** â€” ASP.NET Core entry point. `Program.cs` is the DI composition root.

## Key Patterns

- **Storage key format**: `{UserId}/{FileId}` â€” generated after the file record is created.
- **Presigned URL workflow**: `POST /files` creates metadata and returns a presigned S3 PUT URL. Client uploads directly to S3.
- **Pagination**: `GET /files?skip=0&take=50`. Take clamped to max 200. Sorted by `UpdatedAtUtc` descending.
- **Authentication**: Optional Cognito JWT Bearer, toggled via `Authentication:Enabled` in config.
- **All I/O is async** with `CancellationToken` threaded throughout.
- **File.Id** is nullable on creation (auto-generated by MongoDB's `StringObjectIdGenerator`).
- **REST endpoints**: Separate `POST /files` (create) and `PUT /files/{id}` (update) â€” not upsert.

## Controller Conventions

- Every controller action MUST declare all response codes with `[ProducesResponseType]` attributes.
- Use the typed generic form `[ProducesResponseType<T>(StatusCodes.StatusXXX)]` when the action returns a body (e.g. `[ProducesResponseType<File>(StatusCodes.Status200OK)]`).
- Use the non-generic form for status-only responses (e.g. `[ProducesResponseType(StatusCodes.Status204NoContent)]`).
- Always use `StatusCodes.*` constants rather than integer literals.
- Minimum required codes per action: success code + `400 BadRequest` (if the action can return it) + `401 Unauthorized` (if `[Authorize]`) + `404 NotFound` (if the resource may not exist).

## Test Patterns

**ApiTests** (`src/ProjectDoomsdayServer.ApiTests/`) â€” Fast, no Docker.
- `IClassFixture<CustomWebApplicationFactory>` with NSubstitute mocks for `IFileStorage` and `IFileRepository`.
- Factory tracks state via `SavedFiles`, `DeletedFileIds`, `FileRecords`. Call `factory.Reset()` between tests.

**E2ETests** (`src/ProjectDoomsdayServer.E2ETests/`) â€” Full-stack, requires Docker.
- `ICollectionFixture<E2EInfrastructureFixture>` (shared containers) + per-class `E2EWebApplicationFactory`.
- Testcontainers spins up MongoDB (`mongo:7.0`) and MinIO. Test isolation via `UniqueUserId()` â€” no teardown needed.
- `_appClient` â†’ in-process server; `_rawHttpClient` â†’ MinIO for presigned PUT uploads.

Assertions use FluentAssertions (`response.StatusCode.Should().Be(HttpStatusCode.OK)`).
